2. 전처리  

예정 작업 내역
1. 문서 로드 시 구조화된 텍스트 형식으로 출력
2. 해당 출력 텍스트를 의미기반으로 청킹
3. 해당 전처리 텍스트를 임베딩하여 벡터DB에 저장 
4. 검색 테스트  


- 문서 로드 시 구조화된 텍스트 형식으로 출력
multi-vector 전략필요 
표 이미지 저장 필요 
표 이미지 문장화
postgreSQL -> PGVector 확장 
문서 로드 (DB저장) -> 저장된 데이터 읽어 의미기나 청킹 (DB저장) -> 임베딩 -> PGVector 저장
