● 프로젝트 구조 요약

  src/
  ├── main.jsx                # React 앱 진입점 (ReactDOM 렌더링)
  ├── App.jsx                 # 루트 컴포넌트 (ChatLayout 래핑)
  ├── index.css               # 전역 스타일
  ├── components/
  │   ├── ChatLayout.jsx      # 상태 관리 + 레이아웃 조합
  │   ├── MessageList.jsx     # 메시지 목록 렌더링
  │   ├── MessageItem.jsx     # 개별 메시지 버블
  │   └── ChatInput.jsx       # 텍스트 입력 + 전송 버튼
  └── hooks/
      ├── scrollHook.js       # 자동/수동 스크롤 제어
      └── sendMessageHook.js  # 메시지 전송 + mock 스트리밍

  핵심 컴포넌트 역할

  - ChatLayout — 중앙 컨트롤러. messages와 isTyping 상태를 소유하고, 두 커스텀 훅을 호출하여 스크롤/메시지 로직을 위임한다. 하위 컴포넌트에 props로 데이터와 콜백을 전달한다.
  - MessageList — messages 배열을 순회하며 MessageItem을 렌더링하는 표시 전용 컴포넌트.
  - MessageItem — role에 따라 사용자(오른쪽)/AI(왼쪽) 버블을 구분하여 표시한다.
  - ChatInput — 텍스트 입력을 받아 onSend 콜백으로 상위에 전달한다.
  - useChatScroll — 메시지 추가 시 자동 스크롤, 사용자가 위로 스크롤하면 자동 스크롤 해제, "최근 메세지" 버튼 표시를 제어한다.
  - sendMessageFunc — 사용자 메시지 추가 → 빈 assistant 메시지 생성 → setInterval로 글자 단위 스트리밍 누적. 현재 mock 응답이며, 실제 API 연동 시 이 훅이 교체 대상이다.
